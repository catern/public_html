PL iceberg

Inspired by the Cursed Computer Iceberg, I'm starting my own iceberg of "cursed" PL facts.

Email me ice (facts) to contribute! Or suggestions on what to link to, for the facts I have below!

Here's what I have so far (vaguely in order of increasing obscurity):

  • Curry-Howard correspondence, programs are just a helpful syntax for writing proofs
  • Cursed call/cc facts? Law of the excluded middle? Double negation elimination?
  • Partially evaluating a partial evaluator with respect to a partial evaluator returns a compiler generator
  • All IO is monadic
  • Parallelism is not concurrency
  • Dynamic languages are static languages
  • Curry-Howard-Lambek correspondence, computational trinitarianism
  • Something about codata?
  • Microkernels are an abstraction inversion
  • You can do GC without tags
  • Tracing GC and refcounting are the same thing
  • You can make a compacting memory allocator without moving pointers.
  • You can pretty print functions without their source code.
  • Data is universe-polymorphic functions/very type-theoretically advanced? Function-encoding of data?
  • All computable functions are continuous
  • Function equality is decidable (at some types)
